# Task: Agent Output Persistence and Confirmation

## Description

Implement the missing confirm-to-persist path for high-impact Agent planning outputs. Clarification, stage planning, and task breakdown must become useful project state only after human confirmation.

## Acceptance Criteria

- [ ] Clarification output can update `Project.direction_card` after confirmation.
- [ ] Stage planning output can create/update stages after confirmation.
- [ ] Task breakdown output can create/update tasks after confirmation.
- [ ] Confirm actions record Agent Timeline events with source event ID and reason.
- [ ] Unconfirmed Agent outputs remain proposals and do not silently mutate project state.
- [ ] Backend tests cover confirm-to-persist behavior and invalid confirmation cases.

## Technical Details

- Work in backend services/routes/schemas for Agent confirmation and project/stage/task persistence.
- Reuse existing `AgentEvent` timeline records where practical.
- Keep human-in-the-loop semantics aligned with `docs/TECH-DESIGN.md`.
