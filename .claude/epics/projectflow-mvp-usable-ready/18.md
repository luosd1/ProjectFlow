# Task: Prompt and Schema Quality Hardening

## Description

Improve Agent prompts and output schemas so real model output is actionable, explainable, and grounded in the current WorkspaceState instead of generic fallback text.

## Acceptance Criteria

- [ ] Clarification prompt asks only high-value questions and produces a direction card with problem, users, value, deliverables, boundaries, and risks.
- [ ] Planning prompt generates 3-5 stages with goals, dates, deliverables, and done criteria.
- [ ] Breakdown prompt creates prioritized tasks with due dates, dependencies, acceptance criteria, and `can_cut`.
- [ ] Assignment, push, risk, and replan prompts cite concrete state: skills, availability, blockers, task status, priorities, and deadlines.
- [ ] Schema validation still rejects fabricated member/task/stage references.
- [ ] Tests include realistic messy project fixtures and representative model outputs.

## Technical Details

- Work in `backend/app/agent/prompts.py`, `backend/app/agent/modules`, `backend/app/agent/output_schemas.py`, and tests.
- Keep prompts compact enough for local demo latency.
- Do not weaken validation just to accept poor model output.
