# Task: Frontend Agent Status and Review UX

## Description

Make Agent state visible and honest in the UI. Users must be able to distinguish successful real model output, repaired JSON, fallback templates, and failures. They must also be able to review and confirm high-impact Agent outputs.

## Acceptance Criteria

- [ ] Agent action UI displays `success`, `repaired`, `fallback`, and `failed` distinctly.
- [ ] Fallback output is labeled as fallback and not presented as full AI success.
- [ ] Failed Agent calls show concise cause and retry path.
- [ ] Loading state shows context analysis progress without misleading claims.
- [ ] Direction card, stage plan, and task breakdown outputs can be reviewed before confirmation.
- [ ] Frontend tests cover fallback labeling, failure state, and confirmation flow rendering.

## Technical Details

- Work in `frontend/src/lib/api.ts`, `frontend/src/lib/types.ts`, project dashboard page, and `frontend/src/components/agent` / project dashboard components.
- Keep page components as composition and state wiring; reusable UI belongs in components.
