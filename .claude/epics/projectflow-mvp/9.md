# Task: Action Cards, Check-in, Risk, Timeline, and Export UI

## Description

Build the frontend surfaces for the execution loop: action cards, check-in submission, manual task status updates, risk cards, replan diff, agent timeline, and review summary export.

## Acceptance Criteria

- [x] Personal action cards show task goal, starter advice, completion criteria, due date, and reason.
- [x] Team next action cards are visible from the dashboard.
- [x] Check-in form captures what was done, blocker, and next-cycle availability.
- [x] Manual status update supports not started, in progress, done, and blocked.
- [x] Risk cards show severity, type, evidence, and recommendation.
- [x] Replan diff shows before/after changes and impact.
- [x] Agent timeline shows evidence, decision, action, and fallback events.
- [x] Export UI can request a review summary and display/copy the result.
- [x] Demo reset control is visible where appropriate and clearly scoped to demo data.

## Technical Details

- Put components under `frontend/src/components/agent`, `checkin`, `risk`, and `assignment` as appropriate.
- Keep buttons and controls purpose-specific.
- Do not make the dashboard a passive list view; it must foreground current next action.
- Preserve loading, empty, error, success, and retry states.
