# Task: Guardrails, Monorepo Bootstrap, and Health Checks

## Description

Establish the project structure and minimum runnable frontend/backend skeleton so later tasks have stable boundaries. This task must preserve the existing AGENTS/CLAUDE workspace rules and make the repo safe for iterative implementation.

## Acceptance Criteria

- [ ] Root guardrail docs are present and aligned with the existing workspace instructions.
- [ ] `.gitignore` covers `.env`, SQLite files, backend data, Python caches, virtualenvs, Node modules, and Next build output.
- [ ] `frontend/` is initialized as a Next.js + React + TypeScript + Tailwind project.
- [ ] `backend/` is initialized as a FastAPI + SQLModel + Pydantic project.
- [ ] Backend exposes `GET /api/health`.
- [ ] Frontend local dev server opens a first usable application screen, not a static marketing-only placeholder.
- [ ] Basic README or run instructions point to backend and frontend commands.

## Technical Details

- Use the directory responsibilities defined in `docs/TECH-DESIGN.md`.
- Keep backend routes under `backend/app/api`.
- Keep backend services under `backend/app/services`.
- Keep frontend pages under `frontend/src/app`.
- Keep reusable UI under `frontend/src/components`.
- Keep frontend fetch wrapper and shared types under `frontend/src/lib`.
