# Task: Demo Seed, Reset, Runbook, and Review Export

## Description

Make the demo deterministic and review-ready by adding seed data, reset flow, demo script, runbook, and review summary export.

## Acceptance Criteria

- [x] Seed data creates a realistic 5-6 member student team.
- [x] Seed data includes member skills, availability, interests, and constraints.
- [x] Seed data includes a project idea, deadline, deliverables, resources, stages, tasks, assignments, check-ins, risks, and action cards sufficient for the demo path.
- [x] Demo reset returns the app to a known state without manual database cleanup.
- [x] Review summary export produces a useful Markdown summary.
- [x] `docs/runbook.md` explains local setup, reset, and demo execution.
- [x] `docs/demo-script.md` gives a 5-minute path through the core loop.
- [x] `docs/seed-scenarios.md` documents the blocker or availability-change scenario used for risk/replan.

## Technical Details

- Keep seed logic under `backend/app/seed`.
- Do not mix seed logic into production services.
- Reset must be explicit and demo-scoped.
- Export should summarize product positioning, current state, risks, replanning, and next actions.
