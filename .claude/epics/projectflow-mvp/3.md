# Task: Backend Domain Models and SQLite Persistence

## Description

Implement the core persistent data model for the MVP in SQLModel and wire SQLite session management. This is the foundation for deterministic workflow state and WorkspaceState assembly.

## Acceptance Criteria

- [ ] Models exist for User, Workspace, WorkspaceMembership, Invitation, MemberProfile, Project, ProjectResource, Stage, Task, AssignmentProposal, AssignmentResponse, AssignmentNegotiation, CheckInCycle, CheckInResponse, TaskStatusUpdate, Risk, ActionCard, and AgentEvent.
- [ ] Enum values match the PRD and technical design.
- [ ] Database initialization creates tables locally.
- [ ] Database URL is configurable through environment settings.
- [ ] SQLite database files are kept out of git.
- [ ] Model tests or smoke checks verify table creation and basic insert/read behavior.

## Technical Details

- Use `backend/app/core/config.py` for settings.
- Use `backend/app/core/database.py` for engine/session/table creation.
- Keep persistence-only definitions in `backend/app/models`.
- Do not put request/response schemas in model files.
- Use UUID primary keys unless a simpler local ID is explicitly justified for a demo-only table.
