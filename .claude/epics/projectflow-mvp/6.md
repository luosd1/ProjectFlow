# Task: Frontend Shell, Onboarding, Workspace, and Intake

## Description

Build the first half of the user flow in the frontend: app shell, identity setup, member profile collection, workspace creation/invitation, and project intake.

## Acceptance Criteria

- [ ] App shell provides stable navigation through the MVP flow.
- [ ] Account setup supports lightweight demo identity creation/selection.
- [ ] Member profile wizard collects skills, available hours, preferred role, interests, and constraints.
- [ ] Workspace creation and invite panel are usable in the local demo.
- [ ] Project intake form captures idea, deadline, deliverables, and resource text/link/file metadata.
- [ ] Core UI states include loading, empty, error, success, and retry where applicable.
- [ ] API calls go through `frontend/src/lib/api.ts`.
- [ ] Shared frontend types are defined in `frontend/src/lib/types.ts`.

## Technical Details

- Keep page files as composition and UI state containers.
- Put reusable domain components under `frontend/src/components/onboarding`, `workspace`, `member`, and `project`.
- Use shadcn/ui selectively and avoid heavy marketing-style layout.
- Use mock data only behind a clear API boundary if backend endpoints are not ready yet.
