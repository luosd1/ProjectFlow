# Task: Verification, Tests, and Demo Stability Hardening

## Description

Close the MVP implementation with targeted verification, smoke tests, UI build/lint, and manual demo hardening. This task should catch integration issues without expanding the product scope.

## Acceptance Criteria

- [x] Backend API smoke tests pass.
- [x] Local frontend CORS preflight is covered.
- [x] Agent schema tests pass.
- [x] Assignment reject/swap/finalize tests pass.
- [x] Check-in -> risk -> replan tests pass.
- [x] Frontend lint passes.
- [x] Frontend build passes.
- [x] Manual demo test covers the full flow from account setup to review export.
- [x] Known demo failure points are documented with mitigation or fallback.
- [x] No secrets, database files, build artifacts, virtualenvs, or node modules are tracked.
- [x] Final status report lists completed scope, remaining risks, and exact verification commands.

## Technical Details

- Use `pytest app/tests/ -v` for backend tests.
- Use `npm run lint` and `npm run build` for frontend verification.
- Prefer focused tests for the MVP path over broad low-value coverage.
- Do not bypass failures by commenting tests or adding skip markers unless the reason is documented and accepted.
