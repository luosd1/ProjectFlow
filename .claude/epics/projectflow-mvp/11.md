# Task: Verification, Tests, and Demo Stability Hardening

## Description

Close the MVP implementation with targeted verification, smoke tests, UI build/lint, and manual demo hardening. This task should catch integration issues without expanding the product scope.

## Acceptance Criteria

- [ ] Backend API smoke tests pass.
- [ ] Agent schema tests pass.
- [ ] Assignment reject/swap/finalize tests pass.
- [ ] Check-in -> risk -> replan tests pass.
- [ ] Frontend lint passes.
- [ ] Frontend build passes.
- [ ] Manual demo test covers the full flow from account setup to review export.
- [ ] Known demo failure points are documented with mitigation or fallback.
- [ ] No secrets, database files, build artifacts, virtualenvs, or node modules are tracked.
- [ ] Final status report lists completed scope, remaining risks, and exact verification commands.

## Technical Details

- Use `pytest app/tests/ -v` for backend tests.
- Use `npm run lint` and `npm run build` for frontend verification.
- Prefer focused tests for the MVP path over broad low-value coverage.
- Do not bypass failures by commenting tests or adding skip markers unless the reason is documented and accepted.
