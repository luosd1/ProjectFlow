# Task: Planning and Assignment Dashboard UI

## Description

Build the dashboard surfaces for direction clarification, stage planning, task breakdown, assignment recommendation, assignment response, negotiation, and final assignment confirmation.

## Acceptance Criteria

- [ ] Project dashboard first view clearly shows the current stage and next recommended action.
- [ ] Clarification view displays agent questions and direction card with confirmation.
- [ ] Plan board displays stages, milestones, tasks, dependencies, priorities, and cut/delay markers.
- [ ] Assignment view displays owner, backup owner, reason, and confirmation state.
- [ ] Members can accept or reject assignment proposals.
- [ ] Rejection flow lets a member select a preferred task and provide a reason.
- [ ] Negotiation panel displays one lightweight swap proposal.
- [ ] Final assignment board makes it clear that owner changes happen only after confirmation.
- [ ] All views include appropriate loading, empty, error, and success states.

## Technical Details

- Put components under `frontend/src/components/agent`, `stage`, `task`, and `assignment`.
- Do not encode agent reasoning in static frontend-only data.
- Keep UI types aligned with backend schemas.
- Use restrained motion for assignment distribution and status transitions.
