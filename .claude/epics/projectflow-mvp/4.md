# Task: Core Workspace and Project APIs

## Description

Implement the deterministic APIs and services for lightweight users, workspaces, invitations, member profiles, project intake, resources, stages, tasks, and WorkspaceState reads.

## Acceptance Criteria

- [ ] Users can be created or selected with lightweight identity fields.
- [ ] A workspace can be created with an owner.
- [ ] Invitations can be created and accepted in a demo-friendly way.
- [ ] Members can create/update profiles with skills, availability, role preference, interests, and constraints.
- [ ] A project can be created with idea, deadline, deliverables, and resource metadata.
- [ ] Stages and tasks can be created, listed, updated, and attached to the project.
- [ ] A WorkspaceState endpoint returns the state needed by the Coordinator Agent.
- [ ] API smoke tests cover the main account -> workspace -> profile -> project path.

## Technical Details

- Keep API routes thin and route files domain-oriented.
- Put validation and state mutation rules in services.
- Use Pydantic schemas from `backend/app/schemas`.
- Avoid formal auth; use lightweight `current_user_id` checks where needed.
- The WorkspaceState response should not fabricate members, tasks, or stages.
