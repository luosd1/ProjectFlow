# Task: Agent Output Persistence and Confirmation

## Description

Implement the missing confirm-to-persist path for high-impact Agent planning outputs. Clarification, stage planning, and task breakdown must become useful project state only after human confirmation.

## Acceptance Criteria

- [x] Clarification output can update `Project.direction_card` after confirmation.
- [x] Stage planning output can create/update stages after confirmation.
- [x] Task breakdown output can create/update tasks after confirmation.
- [x] Confirm actions record Agent Timeline events with source event ID and reason.
- [x] Unconfirmed Agent outputs remain proposals and do not silently mutate project state.
- [x] Backend tests cover confirm-to-persist behavior and invalid confirmation cases.
