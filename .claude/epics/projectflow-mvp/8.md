# Task: Assignment, Active Push, Check-in, Risk, Replan Backend Flows

## Description

Implement the backend workflow services and agent endpoints that turn ProjectFlow from planning into active execution: assignment proposal/response/negotiation/finalization, active push, check-ins, risk analysis, replanning, and timeline updates.

## Acceptance Criteria

- [ ] Agent endpoints exist for clarify, plan, breakdown, assign, active push, check-in analysis, risk analysis, and replan.
- [ ] Assignment proposals can be created, accepted, rejected, negotiated, and finalized.
- [ ] Final assignment updates task ownership only after confirmation.
- [ ] Active push creates personal task cards and team next action cards with reasons.
- [ ] Check-in cycles can be created with cadence days.
- [ ] Members can submit what they did, blocker, and next-cycle availability.
- [ ] Manual task status updates record status, progress note, blocker, and availability change.
- [ ] Risk analysis can produce at least deadline, workload, and blocker/dependency style risks.
- [ ] Replan proposal includes before/after, impact, reason, and requires confirmation.
- [ ] Timeline records all agent decisions and fallback events.
- [ ] Tests cover assignment flow and check-in -> risk -> replan.

## Technical Details

- Keep route handlers thin.
- Put workflow rules in services.
- Do not let the agent directly finalize assignment or task owner.
- Risk cards require evidence.
- Replan proposals must be proposals until confirmed.
