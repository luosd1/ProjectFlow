C 方案细化：对话 + 下一步建议流

这版把“流畅”定义成：用户点一个建议后立即进入执行态，Agent 用步骤状态解释正在做什么；生成结果直接回到对话里，确认入口就在结果卡上，不再让用户去页面里猜位置。

建议采用的交互节奏

  1. 顶部只保留当前焦点和待确认数量，不再占据大量空间。
  2. 每轮 Agent 回复最多给 3 个建议选项，主建议高亮。
  3. 点击建议后，先插入用户消息，再立刻显示运行状态。
  4. 运行状态显示“读取签到 / 判断影响 / 生成草案”等短步骤。
  5. 产物卡直接出现在对话中，带确认、修改、查看影响。
  6. 确认后给明确成功反馈，并把下一个建议接上。
