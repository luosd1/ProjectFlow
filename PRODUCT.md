# Product

## Register

product

## Users

大学生项目团队：课程项目、竞赛团队、社团项目。典型场景是 3-8 人的学生小组，需要在一个学期内完成一个项目，成员技能和时间不固定，缺乏项目管理经验，需要工具帮助分配任务、跟踪进度、识别风险，并由 AI 主动推进项目流程。

## Product Purpose

ProjectFlow 是一个带主动 AI Agent 的大学生团队项目管理工具。它持续回答三个问题：项目该往哪走？谁适合做什么？哪些有风险、计划是否需要调整？

核心流程：项目录入 → AI 澄清方向 → 阶段规划 → 任务拆解 → 智能分工 → 执行推进（行动卡、签到）→ 风险监控 → 重排建议 → 导出复盘。

## Brand Personality

简洁、智能、高效

- **简洁**：信息层级清晰，不堆砌功能，渐进式展示
- **智能**：AI 主动推进，不是被动工具，能推断的不让用户填
- **高效**：一步完成的不拆成三步，反馈引导行动而非只报告问题

参考：
- https://app.lobehub.com/ — 清晰的信息架构和现代化的组件设计
- https://elevenlabs.io/ — 精致的排版和动效节奏

## Anti-references

- 避免 SaaS 奶油风：不要暖米色背景 + 圆角卡片 + 大阴影的千篇一律模板感
- 不要渐变文字装饰
- 不要大圆角（卡片 ≤ 16px，按钮/标签可用 pill）
- 避免过度手绘/涂鸦风格插画
- 避免营销 buzzword 文案（streamline / empower / supercharge 等）

## Design Principles

1. **为目标设计，不为功能设计**：先问「用户要完成什么」，再决定怎么实现
2. **系统承担复杂性**：能自动化的不手动，能推断的不让用户填
3. **渐进式展示**：先给核心，细节按需展开，不要一次性甩出所有选项
4. **反馈引导行动**：不要只报告问题，要引导下一步
5. **保持克制**：不必要的副标题、小字注释、装饰元素一律去掉

## Accessibility & Inclusion

- WCAG 2.1 AA
- 支持 `prefers-reduced-motion`
- 确保颜色对比度 ≥ 4.5:1（正文）、≥ 3:1（大文字）
